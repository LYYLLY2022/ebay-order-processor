# eBay 订单处理系统

自动化处理从 eBay 导出的 XML 格式订单报告，实现数据清洗、转换、汇总和报表生成。

## 功能特性

### 第一阶段：文件上传与校验
- **多文件上传**：支持拖拽或点击上传多个 XML 文件
- **合法性过滤**：自动忽略非 .xml 后缀文件
- **去重机制**：基于文件名+文件大小自动去重，防止重复上传

### 第二阶段：XML 解析与表头定位
- **标准格式支持**：支持 Excel XML 2003 格式
- **动态表头识别**：自动识别并验证必需字段（eBay帐号、站点、货币、总额、购买日期）
- **错误检测**：缺少必需字段时自动报错提示

### 第三阶段：核心数据清洗与转换
- **站点名称标准化**：
  - Germany → DE
  - Australia → AU
- **无效货币过滤**：只保留白名单货币（GBP、USD、AUD、EUR）
- **金额提取**：将总额字符串转换为浮点数
- **汇率折算**：根据配置的汇率计算折算后的 USD 金额
- **时区转换**：将中国时间（UTC+8）转换为 UTC 时间（减去 15 小时）

### 第四阶段：分组与汇总
- **联合主键生成**：按"购买日期(UTC) | eBay帐号 | 站点 | 货币"分组
- **数据累加**：自动合并相同主键的记录，累加原始货币总额和折算后 USD 总额
- **时间排序**：按购买日期升序排列结果

### 第五阶段：Excel 报表生成与导出
- **标准报表格式**：包含购买日期、eBay帐号、站点、货币、原始货币总额、总额(折算USD) 六列
- **数据统计**：显示处理文件数、总数据行数、分组记录数、过滤行数
- **合计与注释**：
  - 底部自动添加合计行（仅计算折算USD总额）
  - 记录时区转换规则（减去 15 小时）
  - 记录使用的具体汇率
  - 记录报表生成时间

## 使用方法

### 1. 上传文件
- 点击上传区域或拖拽 XML 文件到指定区域
- 系统会自动校验文件格式并去重

### 2. 设置汇率
- 在汇率设置区域修改各货币对 USD 的汇率
- 默认汇率：GBP=1.33, USD=1.00, AUD=0.70, EUR=1.15

### 3. 开始处理
- 点击"开始处理"按钮
- 系统会自动解析、清洗、转换和汇总数据
- 实时显示处理进度

### 4. 导出报表
- 处理完成后点击"导出 Excel 报表"按钮
- 系统会自动下载生成的 Excel 文件

## 技术栈

- **前端框架**：原生 JavaScript
- **UI 设计**：现代渐变风格，响应式布局
- **Excel 导出**：使用 SheetJS (xlsx) 库
- **文件解析**：浏览器原生 DOMParser

## 注意事项

1. **文件格式**：仅支持 Excel XML 2003 格式的 .xml 文件
2. **必需字段**：文件必须包含以下字段：eBay帐号、站点、货币、总额、购买日期
3. **货币限制**：只处理 GBP、USD、AUD、EUR 四种货币，其他货币将被过滤
4. **时间格式**：购买日期应为 YYYYMMDD HH:mm:ss 格式，默认为中国时间（UTC+8）
5. **浏览器要求**：建议使用现代浏览器（Chrome、Firefox、Edge 等）
